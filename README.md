# Живой трекинг онлайна игр

Backend + фронтенд для трекинга реального онлайна игр Steam и Roblox.
**Принцип проекта: ноль фейковых данных.** Если backend не подключён —
сайт честно показывает пустое состояние с объяснением, а не случайные числа.

## Архитектура

```
game-tracker/
├── backend/
│   ├── main.py        # FastAPI: REST + WebSocket + добавление игр + сравнение
│   ├── collector.py   # опрос Steam/Roblox, сбор топ-чарта одним запросом
│   ├── db.py            # SQLite: pinned-игры, кэш названий, статистика
│   └── games_list.py   # стартовый набор игр (seed)
├── frontend/
│   └── index.html      # SPA: список, страница игры, сравнение, форма добавления
└── requirements.txt
```

## Как теперь получается "топ-1000 реальных игр" (Steam)

Раньше: дёргать `GetNumberOfCurrentPlayers` по каждой игре отдельно — это
нереалистично для тысячи игр. Теперь: один запрос к
`ISteamChartsService/GetGamesByConcurrentPlayers` — это тот же
недокументированный, но открытый эндпоинт, который питает страницу
store.steampowered.com/charts/mostplayed. Он отдаёт сразу рейтинг игр **с их
текущим онлайном** — никаких дополнительных запросов на игру не нужно.

Поскольку Valve не публикует точную схему ответа, парсинг в `collector.py`
написан защитно (пробует несколько вероятных названий полей) и при первом
успешном вызове пишет в лог пример первой записи — если поля называются
иначе, это будет видно в логах Railway за 10 секунд и легко исправить одной
строкой.

Названия игр для всех appid реальные — раз в сутки backend загружает полный
`GetAppList` (все Steam-игры с названиями) и кэширует в SQLite, поэтому
каждая игра из чарта подписывается настоящим именем, а не "Steam App #730".

Иконки — публичный CDN `cdn.cloudflare.steamstatic.com`, доступны мгновенно
по appid, без какого-либо API-вызова.

## Почему Roblox не дотягивает до 1000 автоматически

У Roblox **больше нет публичного и стабильного API для топ-N игр** —
легаси `games.roblox.com/v1/games/list` официально сломан (подтверждено в
форуме разработчиков Roblox, актуально на 2025–2026). Поэтому Roblox-игры
трекаются только из стартового набора и того, что добавят пользователи через
форму на сайте — зато точечный запрос онлайна по известному `universeId`
работает надёжно и даёт настоящие живые цифры.

## "Добавить игру" — сохраняется для всех

Кнопка "+ Добавить игру" вызывает `POST /api/games/add`. Backend:
1. Проверяет ID через настоящий публичный API (Steam или Roblox) —
   несуществующий ID не добавится.
2. Для Roblox: если вставлен `placeId` из ссылки на игру, backend сам
   конвертирует его в `universeId` через `apis.roblox.com/universes/v1/places/...`.
3. Сохраняет игру в SQLite с флагом `pinned=1` — она навсегда попадает в
   список опроса, видна **всем** посетителям сайта, не только тебе.

## Сравнение игр

В списке у каждой игры — чекбокс. Выбери 2–3 игры → внизу появляется панель
"Сравнить" → переход на страницу с одним графиком (до 3 линий разных цветов),
легендой и таблицей метрик (текущий, пик 24ч, средний 24ч, пик за всё время).
Эндпоинт: `GET /api/compare?games=steam:730,steam:570&hours=24`.

## Исправлено по фидбеку

- **Иконки**: Steam — настоящие обложки с CDN; Roblox — настоящие иконки,
  которые backend получает через `thumbnails.roblox.com`.
- **Сплющенный график**: раньше SVG растягивался нерав­номерно
  (`viewBox` не совпадал с реальным размером контейнера → искажение по X).
  Теперь перед отрисовкой код измеряет фактическую ширину контейнера
  (`getBoundingClientRect`) и подставляет её в `viewBox` 1:1 — искажения нет
  независимо от ширины экрана. Пересчитывается и при ресайзе окна.
- **Фейковый онлайн**: весь генератор случайных чисел и синтетические игры
  удалены. Без backend список пуст и явно говорит об этом.
- **1000 реальных игр**: см. раздел про `GetGamesByConcurrentPlayers` выше.
- **Добавление игр + общая персистентность**: реализовано через SQLite на
  сервере, см. раздел выше.
- **Сравнение игр**: реализовано, см. раздел выше.

## Запуск

```bash
cd game-tracker
pip install -r requirements.txt
cd backend
uvicorn main:app --reload --port 8000
```

Открой http://localhost:8000

## Деплой на Railway

1. Запушить `game-tracker` в GitHub.
2. Railway → New Project → Deploy from GitHub repo.
3. Start command: `cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT`
4. Подключить Railway Volume к `backend/data/` — иначе SQLite слетит при
   редеплое (как и раньше).
5. Опционально: переменная окружения `STEAM_API_KEY` — если чарт-эндпоинт
   или GetAppList начнут требовать ключ (Valve иногда меняет правила), просто
   добавь её, код уже готов её подхватить.
