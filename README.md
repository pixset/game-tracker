# PULSE.GG — живой трекинг онлайна игр

MVP сервиса, который собирает данные о текущем количестве игроков
в играх Steam и Roblox, хранит историю и отдаёт топ-10 + страницу
статистики по каждой игре в реальном времени (через WebSocket).

## Архитектура

```
game-tracker/
├── backend/
│   ├── main.py          # FastAPI: REST + WebSocket, отдаёт фронтенд
│   ├── collector.py     # опрос Steam/Roblox API, раз в N секунд
│   ├── db.py             # SQLite: схема, запросы (top10, история, статистика)
│   └── games_list.py     # список трекаемых игр (стартовый набор)
├── frontend/
│   └── index.html        # одна страница: лента + топ-10 + спарклайны
└── requirements.txt
```

Поток данных:
1. `collector_loop` каждые 60 секунд опрашивает Steam (`GetNumberOfCurrentPlayers`,
   не требует API-ключа) и Roblox (`games.roblox.com/v1/games`, батчем по universeId).
2. Каждый замер пишется в `snapshots` (источник, id игры, время, кол-во игроков).
3. `broadcast_loop` каждые 5 секунд берёт свежий топ-10 из БД и пушит всем
   подключённым клиентам по WebSocket `/ws`.
4. Фронтенд сначала пробует достучаться до `/api/top10` — если backend живой,
   переключается на реальные данные, иначе показывает demo-режим со случайным
   блужданием цифр (чтобы дизайн можно было оценить без деплоя).

## Запуск локально

```bash
cd game-tracker
pip install -r requirements.txt
cd backend
uvicorn main:app --reload --port 8000
```

Открой http://localhost:8000 — сайт и API на одном порту.

## Деплой на Railway (как твои предыдущие боты)

1. Запушить папку `game-tracker` в GitHub-репозиторий.
2. В Railway: New Project → Deploy from GitHub repo.
3. Start command:
   ```
   cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT
   ```
4. **Важно**: подключить Railway Volume к `backend/data/` — иначе SQLite-база
   слетит при каждом редеплое. Это та же логика, что у тебя была с
   персистентными данными в медиа-боте.
5. Никаких секретных переменных не нужно — оба API публичные и бесплатные.

## Масштабирование до "всех игр"

Сейчас список игр — захардкоженный топ-15 Steam + 1 Roblox-игра, чтобы сразу
было видно живые данные. Чтобы расти до реально "всех игр":

**Steam (тысячи игр автоматически):**
- Раз в сутки дёргать `ISteamApps/GetAppList/v2` — отдаёт ВСЕ appid Steam.
- Не у каждой игры есть онлайн (инди без чарта = result:0) — collector уже
  такие тихо пропускает, ничего менять не нужно.
- При тысячах appid важно батчить запросы с лимитом параллельности (уже есть
  `Semaphore(10)` в `collector.py` — просто увеличить, если Steam не режет).
- Названия/обложки игр одним вызовом не получить массово — либо кэшировать
  через `store.steampowered.com/api/appdetails` (жёсткий рейт-лимит, ~1 запрос
  в секунду), либо взять готовый дамп из SteamSpy.

**Roblox (поиск топовых игр):**
- Эндпоинт `games.roblox.com/v1/games/list?sortToken=...` отдаёт каталог по
  популярности — можно подтягивать ID автоматически, а не вручную.

**Как добавить Roblox-игру вручную (пока каталог не подключён):**
1. Взять `placeId` из URL игры: `roblox.com/games/<placeId>/...`
2. `GET https://apis.roblox.com/universes/v1/places/<placeId>/universe` → `universeId`
3. Добавить `{"universe_id": "...", "name": "..."}` в `ROBLOX_GAMES`.

**Когда вырастет нагрузка / объём данных:**
- SQLite → Postgres (Railway даёт одним кликом) — запросы в `db.py` написаны
  на простом SQL, переезд почти без переписывания.
- Снэпшоты старше N дней — агрегировать в часовые/дневные бакеты, чтобы не
  раздувать таблицу `snapshots` бесконечно.

## Что уже реализовано сверх базового MVP

- **Страница игры** (`#/game/steam/730`) — подробная статистика (пик/среднее за
  24ч и за всё время, число замеров) + график динамики с переключением
  диапазона 24ч / 7д / 30д. Роутинг через hash, страница открывается напрямую
  по ссылке — можно делиться конкретной игрой.
- **Фильтры**: по источнику (Все/Steam/Roblox) и по размеру выборки
  (Топ 10/100/1000) — работают мгновенно на клиенте без перезапроса к серверу,
  так как WebSocket изначально присылает расширенный лидерборд.
- **Поиск по названию** — мгновенная фильтрация списка.
- **Избранное (★)**: можно отметить игры звёздочкой — список сохраняется в
  `localStorage` браузера, есть фильтр "Только избранное". Полезно для тех,
  кто следит за конкретными играми, а не топом целиком.
- **"Растут быстрее всех"** — отдельная строка над списком с тремя играми,
  у которых сильнее всего вырос онлайн за последний замер. Помогает заметить
  внезапный всплеск (например, после обновления игры или вирального видео).
- **Индикатор LIVE/DEMO** в хедере — честно показывает, подключен ли реальный
  backend или сайт работает на тестовых данных.

## Идеи для дальнейшего развития

- **Telegram-бот компаньон**: подписка на игру → алерт при резком
  падении/росте онлайна (твой стек aiogram + общая БД — ничего нового учить
  не придётся).
- **Сравнение игр**: страница с 2-3 графиками онлайна на одной шкале времени.
- **Публичный API**: отдавать `/api/top` как открытый JSON-эндпоинт —
  другие разработчики/блогеры будут на него ссылаться, это органический трафик.
- **Синхронизация избранного между устройствами**: сейчас оно живёт только в
  localStorage браузера; при появлении аккаунтов можно перенести в БД.
